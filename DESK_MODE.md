# 데스크 안내 모드 (Desk Interpretation Mode) — 기획·설계 문서

국제선 안내 데스크에서 **안내원 ↔ 외국인 여행객**이 마주 보고 실시간 대면 통역하는 새 모드.
기존 "한 명이 설명/온라인 회의" 시나리오와 별개의 **독립 메뉴**로 추가한다.

> 선행 결정 요약(대화에서 확정된 것)
> - 마이크/캡처: **호스트 쪽 1개**(여객 태블릿은 표시 전용, 입력 없음)
> - **세션 시작 = 여객 터치**(만국 공통 동작, 읽을 필요 없음). 터치 전엔 마이크/soniox OFF → **비용 0 · 직원 잡담 안 들음 · 오발동 없음**
> - 발화감지: **VAD(입력 음성량 임계값)** — PTT 안 씀. 임계값은 세부설정으로 조절
> - **TTS 포기** → 텍스트 전용 → 스피커 출력 없음 → **피드백 루프 원천 소멸**
> - 언어 흐름: 터치 후 **상대 언어 자동확정(B)** → soniox `two_way` 전환 (A=ko 자명, 국기 선택 불필요 — 자동감지 유지)
> - 언어 후보: **취항국 언어로 한정**(감지 정확도↑)
> - 7초 무발화 → **세션 종료(초기화)**, 단 **대화기록은 보존**
> - 여객 초기화면: "당신의 언어로 말해보세요"를 **여러 언어로**
> - 태블릿: **안드로이드 + Chrome 키오스크 권장**(iPad는 오디오 제약 큼)

---

## 1. 목표 / 비목표

**목표**
- 안내원과 외국인이 **버튼 없이** 번갈아 말하면 양쪽 화면에 각자 언어로 즉시 표시.
- 손님이 떠나면 **자동으로 다음 손님 대기 상태**로 리셋.

**비목표**
- 음성 출력(TTS) 없음. 4개국어 동시번역(whisper) 없음. 회의/강연 기능과 무관.

---

## 2. 하드웨어 구성

```
        [ 안내 데스크 ]
 안내원 ──── 마이크(호스트측 1개) ──── 외국인
   │                                      │
 호스트 태블릿                        여객 태블릿
 (캡처+표시, 로그인)                  (표시 전용, QR 접속)
```

- 마이크는 호스트측에 물림. **검증 필요**: 호스트 마이크가 데스크 건너편 여객 목소리를 소음 속에서 충분히 잡는가(§9 체크리스트).
- 스피커 불필요(TTS 없음).

---

## 3. 아키텍처 — 기존 자산 재사용 매핑

새 파이프라인을 만들지 않는다. **soniox `two_way`** 가 이미 양방향 엔진이다.

| 데스크 현장 | 기존 역할/코드 | 비고 |
|---|---|---|
| 안내원 태블릿 | `handleHost` (인증 host) | soniox 파이프라인 소유, 마이크 캡처 |
| 여객 태블릿 | `handleViewer` (QR 공개) | 표시 전용. **PTT(`startTalkPipeline`) 미사용** |
| 번역 엔진 | `runSoniox` (`server.js`) | `two_way` 모드 재사용 |
| 캡처/VAD | `startRecorder` (`audio.js`) | `TH=0.015` 임계값 노출 필요 |
| 양방향 설정 | `sxMode/sxA/sxB`, `roomCfg` | 동적 재설정 필요(§5) |
| 유휴 종료 | `bumpIdle/idleClose`, `IDLE_LIMIT_MS` | 7초로 값만 변경 + 종료 시 동작 확장(§6) |

- 인증 모델도 그대로 맞음: 안내원은 본인 계정 로그인(host), 여객 태블릿은 QR 공개 접속(viewer).
- **새 세션 타입**: 기존 `pipeline ∈ {whisper, translate, deepgram, soniox}` 에 개념적으로 "desk" 추가(내부적으론 soniox two_way + 데스크 생명주기). 세션 생성 모달에 별도 메뉴로 노출.

## 4. 세션 생명주기 (핵심 흐름)

```
[대기]  여객 화면: "터치해서 시작"(다국어·깜빡임)  ·  마이크/soniox OFF (비용 0, 잡담 안 들음)
   │
   │ ① 여객(또는 직원)이 뷰어 터치  ← 세션 게이트 (만국 공통 동작)
   ▼
[감지 모드]  마이크/soniox ON. 전사+언어식별만(번역X, 저렴) 또는 one_way→ko
            A=ko 자명. 호스트·여객 누가 먼저 말해도 됨.
   │
   │ ② 첫 비-한국어 발화 → 언어식별로 X 자동감지(디바운스)
   ▼
[B 확정]  soniox 를 two_way(ko ↔ X) 로 재연결
          여객 화면 표시언어 = X 자동 설정 (국기 선택 불필요)
   │
   │ ③ 진짜 양방향 대화: 안내원 ko→X, 여객 X→ko (VAD 자동 커밋)
   ▼
[종료]  "종료" 터치 또는 외국어 무음 7초(설정값)
        → 기록 저장(append) → 화면 리셋 → 마이크/soniox OFF → [대기]
```

> **터치는 세션을 "여는" 게이트일 뿐, 언어 선택이 아니다.** 언어는 터치 후에도 자동감지로 확정(외국인이 국기에서 자기 언어를 찾을 필요 없음). 터치가 비용·잡담·오발동을 막고, 자동감지가 무설정 경험을 유지 — 둘을 함께 가져간다.

**언어 자동확정 설계(난관)**
- soniox `two_way`는 스트림 시작 전에 `language_a/b`가 필요 → 터치 후 2단계로 우회.
  1. 감지 모드: `enable_language_identification: true` + `language_hints: [취항국 언어 후보]` (전사+LID, 번역은 생략 가능 → 저렴). 또는 `one_way, target_language: ko`로 첫 외국어 문장을 바로 번역.
  2. X가 신뢰수준으로 확정되면 soniox 재연결 → `two_way { language_a: ko, language_b: X }`.
- 첫 외국어 발화는 단방향(외국어→ko)으로 처리됨. 안내원 답변 양방향은 X 확정 후.
- 국기 선택은 자동감지 실패/오인 시 **수동 폴백**으로만 제공.
- **디바운스/신뢰도**: 노이즈·한 토큰 오인으로 잘못 잠그지 않도록 최소 토큰 수/일관성 확인 후 잠금. 잘못 잠기면 수동 "언어 다시 감지" 폴백.
- `roomCfg`(현재 sxA/sxB/sens 등 보관)에 desk 상태(phase: waiting|detecting|locked, lockedLang) 추가.

### 4-1. 엣지 케이스: 한국인 방문객 (둘 다 한국어)

이 설계는 **"언어 = 방향"** 으로 라우팅한다(단일 마이크, 언어로 자동 분기). 따라서 **양쪽 다 한국어면 라우팅할 상대 언어가 없다.**

**처리 원칙: 통역하지 않고 자연스럽게 빠진다.**
- **B(외국어)는 "확신 있는 비-한국어" 감지에서만 잠근다.** 한국어만 들어오면 B 미확정 → two_way 미발동 → 통역 안 함.
- 안내원은 평소처럼 한국어로 응대, 태블릿은 대기 상태 → 7초 후 리셋. 한국인 손님은 별도 처리 불필요(외국어 태블릿을 보지 않음).

**디테일 2가지**
1. **직전 손님 언어락 잔존**: 이전 X가 안 풀린 채 한국인이 오면 안내원 한국어가 X로 헛번역되어 여객 화면에 뜸(무해하나 지저분). → **손님 교체(공백/리셋)에서 언어락을 반드시 해제.**
2. **같은 언어 두 명은 본질적으로 구분 불가**(단일 마이크+언어 라우팅). 단 통역이 불필요한 상황이라 "통역 대기"로 두면 됨 — 한계가 아니라 정상 동작.

**선택 UX**: 한국어만 감지될 때 호스트 화면에 "한국어 감지 — 통역 대기" 소형 상태표시 → 안내원이 통역 미작동을 인지.

### 4-2. 엣지 케이스: 직원 잡담 / 상시 마이크

상시 마이크라면 직원끼리의 한국어 대화가 종일 들어와 ① 로그 오염 ② 비용 ③ 리셋 불가 ④ 화면 노이즈를 일으킨다.

**1차 방어 = 터치 게이트(§4 ①).** 터치 전엔 마이크/soniox가 OFF라 **직원 잡담을 애초에 안 듣는다.** 이것만으로 위 4가지 대부분이 해소되고 **유휴 비용이 0**이 된다.

**2차(터치 후) = "외국어 활동" 기준으로 유지/종료.** 터치됐는데 상대가 한국인이거나(§4-1), 응대 중 직원이 한국어로 잠깐 떠드는 경우 대비:
- **종료 = "외국어 무음" 7초**(전체 무음 아님). 외국인이 없으면 응대 종료로 간주 → 리셋 + 언어락 해제.
- **B 확정 = 확신 있는 비-한국어**(다토큰·일관성). "OK/gate" 단발 외래어는 무시(디바운스) → 오발동 방지.
- **로그 = 외국어가 한 번이라도 잡힌 대화만**(한국어만이면 저장 안 함).

**비용**: 터치 게이트로 유휴=0. 응대 중 추가 절감은 "감지 단계는 번역 없이 전사+LID만(저렴) → 외국어 확정 후 번역 켬"(§4 1단계).

**남는 한계(정직)**: 응대 도중 직원이 동료에게 한국어로 말하면 외국어로 번역되어 화면/로그에 섞일 수 있음. 단일 마이크 구조의 본질적 한계, 빈도 낮고 무해.

## 5. 동적 언어 재설정

- 현재 `sxA/sxB`는 **세션 생성 시 고정**, `startTalkPipeline`은 `L4=['ko','en','ja','zh']`로 제한.
- 데스크 모드는 **세션 도중 B를 바꿔야** 함(손님마다 다름).
- 필요한 변경:
  - viewer→server 또는 host→server 제어 메시지로 "언어 재설정" 트리거 → soniox 재연결.
  - 언어 후보 목록을 **취항국 언어 집합**으로 확장(L4 제한 해제). 단 **soniox two_way 지원 언어쌍**만 허용(§9에서 검증).
- 종료 시 B를 풀고 다시 감지 모드로 되돌림.

## 6. 종료 & 기록 보존

- 7초(설정값) **외국어 무음** 시(전체 무음이 아니라 §4-2 기준): 기존 `idleClose` 메커니즘 재사용하되 동작을 "OA 종료"가 아니라 **"대화 마감 + 리셋 + 언어락 해제"** 로 확장.
- 마감 시: 현재 `session.items`(이번 대화)를 **로그로 append 저장** → 화면용 `items`는 비움 → soniox 감지 모드로 재시작.
- 자료구조: 한 데스크 세션 안에 여러 "대화(conversation)"가 누적되는 형태, 또는 대화별 별도 레코드.

> ⚠️ **개인정보 주의**: 데스크 통역 로그엔 개인정보가 섞일 수 있음(앞서 "휘발성/개인정보 배제" 방향과 상충). 권장 절충:
> - 기록은 남기되 **자동 파기(N일 후 삭제)** 또는 **익명화/통계만** 옵션 제공.
> - "남기되 안전하게"를 기본 정책으로.

## 7. UI 스펙

**공통 컨셉**: 큰 글씨, 현재 발화 1개를 크게(하단), 직전 1~2개만 위로 갈수록 흐리게(opacity 페이드). 군더더기 제거.

**화자 색 구분(언어로 감지)**
- 한국어(안내원측) = 색 A(예: `primary`), 외국어(여객측) = 색 B(보조색). `side`가 아니라 **감지된 언어**로 색 결정.
- 호스트·여객 **양쪽 화면 모두 동일 색 규칙** → 누가 한 말인지 일관되게 인지.

**호스트 태블릿**
- 한국어 위주로 크게. 현재 발화 강조 + 직전 1~2개 페이드.
- 하단/구석에 상태: 대기 / 감지중 / `ko↔X 연결됨` / 종료-리셋.
- 세부설정 진입(§8).

**여객 태블릿**
- 대기화면: 큰 **"터치해서 시작"** 버튼(전체화면급) + 손가락 탭 아이콘 + **다국어 순환/동시 표기**(Tap to start / 開始 / Touch / Bắt đầu …). 읽지 못해도 터치 동작은 만국 공통, 안 되면 직원이 대신 탭.
- 터치 후: "말해보세요"(다국어) → 첫 발화로 언어 자동감지.
- 대화 시작되면: 자국어(X)로 큰 글씨, 동일한 화자 색 규칙, 직전 1~2개 페이드. 하단에 "종료" 버튼.

**구현 베이스**: 호스트는 `TranslateView.jsx`, 여객은 `mobile.html`을 베이스로 데스크 전용 레이아웃 추가(기존 화면 건드리지 않고 모드 분기).

## 8. 세부설정값 (Advanced Settings)

| 설정 | 매핑 | 기본값 | 비고 |
|---|---|---|---|

| 엔드포인트 민감도 | `sxSens`(endpoint_sensitivity) | 기존값 | **이미 고급설정 다이얼로그 존재**(커밋 86eb681) |

| 엔드포인트 지연조정 | `sxLatency` | 0 | 〃 |
| 언어 후보(취항국) | `language_hints` | 취항국 집합 | 감지 정확도 |

## 9. 검증 우선 항목 (구현 전 테스트)

설계의 토대를 거는 부분. **이걸 먼저 깨야 나머지가 의미 있음.**

1. **soniox `two_way` 언어쌍 지원** ⭐ — **[문서 확인 완료]** two_way는 지원 언어 중 임의의 두 개 가능(60+ 언어, 3,600쌍, 쌍 제한 명시 없음). 취항국 언어 중 **지원**: ko·en·ja·zh·vi·th·tl(타갈로그)·id·ru·ms(말레이). **미지원**: 몽골어, 광둥어. → **몽골 노선 승객은 자동감지/번역 불가 → 영어 등 폴백 정책 필요.** 언어식별+화자분리+엔드포인트는 번역과 동시 사용 가능(문서 명시). 라이브 확인: `test/soniox-pairs.mjs`(키 필요).
2. **첫 발화 언어감지 속도/정확도** — **[라이브 검증 완료]** en·ja·zh·id 4개 언어 음성(macOS `say` 합성)으로 end-to-end 확인:
   - 첫 언어감지 **≈ 1.6~1.7초** (en 1.58s / id 1.60s / ja 1.73s / zh 1.58s) → **디바운스/engage 임계의 기준값**.
   - 전사·한국어 번역 품질 양호(공항 질문 문장 기준).
   - 검증법: `detect 모드 = one_way→ko + 언어식별`이 **언어감지 + 첫 질문의 한국어 번역을 동시에** 제공 → §4 1단계 설계 그대로 유효(첫 외국어 발화 = 직원용 한국어로 바로 번역, 이후 two_way 전환).
   - 잔여 검증: 실제 마이크·공항 소음·여러 언어 혼재 환경(실사용 단계).
3. **호스트측 단일 마이크의 여객 픽업** — 데스크 건너편 목소리를 공항 소음 속에서 잡는가? (안 되면 탁자 회의 마이크로 변경 검토)
4. **VAD 오작동** — 공항 소음에서 빈 발화 커밋 빈도. `TH`/민감도 튜닝.
5. **키오스크 현실성** — 마이크 1회 권한허용 유지, Wake Lock(화면 안 꺼짐), 전체화면 고정, 탭 throttling 회피. (안드로이드/Chrome 기준)

## 10. 작업 목록 (구현 시)

대부분 재사용, 신규는 다음 정도:

1. **새 "데스크 안내 모드" 세션 타입/메뉴** — 생성 모달에 추가, soniox two_way + 데스크 생명주기 바인딩.
2. **터치 게이트** — 여객 뷰어 대기화면(다국어 "터치해서 시작"). 터치→host에 "응대 시작" 제어 메시지→마이크/soniox 연결, "종료"/유휴→해제. 터치 전엔 캡처/엔진 미연결(비용 0).
3. **언어 감지→two_way 동적 재설정** — `runSoniox` 2단계화(detecting→locked), `roomCfg` 상태 확장, 재연결 트리거.
4. **7초(외국어 무음) 종료→기록 저장→리셋 롤오버** — `idleClose` 확장, 대화 append 저장, 화면 초기화, 언어락 해제.
5. **호스트 PTT 불필요** — 단, VAD 임계값(`TH`) 설정 노출.
6. **언어 후보 확장** — `L4` 제한 해제 → 취항국 집합(soniox 지원 검증된 것만).
7. **데스크 UI** — 호스트/여객 대형 텍스트 + 언어별 색 + 페이드 히스토리 + 여객 터치 대기화면.
8. **기록 보존/자동파기 정책** — 저장 + 만료 삭제.
9. **키오스크 보강** — Wake Lock, 권한 유지, 전체화면.

## 11. 웹앱 한계 & 대응 (참고)

| 한계 | 영향 | 대응 |
|---|---|---|
| 마이크 권한(제스처 필요) | 키오스크 자동실행 시 재요구 | 1회 사전 허용, 동일 origin 유지 |
| 화면 잠자기/탭 throttling | 오디오 중단 | Wake Lock API + 키오스크 설정 |
| 백그라운드 동작 불가 | 탭 닫히면 멈춤 | 키오스크 런처 전체화면 고정 |
| ScriptProcessorNode(현 코드) | deprecated | AudioWorklet 이전(권장, 비긴급) |
| iPad Safari | 오디오 제약 큼 | **안드로이드+Chrome 권장** |
| 네트워크 의존 | 끊기면 중단 | soniox=클라우드라 동일. 오프라인 안내문 폴백 |

**결론**: 안내원이 상주하는 유인 데스크라면 웹앱으로 v1 충분. 네이티브는 무인 대량배포/딥 하드웨어 제어 시에만 검토.

## 12. 구현 상태 (MVP)

**1차 구현 완료 (엔진 + 최소 연결)** — `node --check` 통과. *주의: 이 작업환경엔 npm/node_modules가 없어 앱 빌드·실행 검증은 못 함 → `npm start`로 실기기 검증 필요.*

- **서버 `runDesk()`** (`server.js`): pipeline `desk`. detect(one_way→ko + 언어식별) → 외국어 첫 문장 endpoint에서 그 언어로 lock → two_way(ko↔X) 재연결. 외국어 무음 `deskIdle`(기본 7초) → 대화 종료(items를 `session.deskLog`에 보존 후 화면 리셋 `desk-reset`/`snapshot:[]`) → 다시 감지. 한국어만 들리면 lock 안 함(§4-1/4-2). TTS·화자분리 없음.
- **와이어링**: createSession `desk` 허용(langs=['ko']), upgrade 파라미터 `deskLangs`·`deskIdle`, dispatch 분기.
- **호스트 UI** (`TranslateView.jsx`): 세션생성에 "데스크 안내" 옵션, 기존 번역뷰 재사용(dispLang=ko 고정 → 외국인=한국어번역·안내원=원문 모두 한국어로 표시), desk용 컨트롤 정리, `desk-reset` 클리어, 고급설정(민감도) 재사용.
- **뷰어 UI** (`mobile.html`): desk 분기 — 외국인 자기 언어(sxInfo.b) 자동 표시, 감지중 라벨, `desk-reset` 클리어. 입력 없음(표시 전용).
- **audio.js**: desk 쿼리 파라미터(sxSens/MaxDelay/Latency, deskLangs, deskIdle).

**검증 방법** (사용자): `npm install && npm start` → 로그인 → 새 세션 "데스크 안내" 생성 → 시작(마이크) → 한국어로 인사 후 외국어로 질문 → 자동 lock·양방향 확인. QR로 폰 접속 시 외국인 언어로 표시. 7초 외국어 무음 → 리셋 확인.

**아직 안 한 것 (다음 단계)**:
- 전용 데스크 UI: 큰 글씨, **언어별 화자 색 구분**, 직전 1~2개 페이드 (현재는 기존 채팅 레이아웃 재사용).
- **터치 시작 게이트**(여객 터치→호스트 캡처 시작): 현재는 호스트가 직접 시작. cross-device 신호 필요.
- 세부설정 UI(deskLangs 취항국 선택, deskIdle 슬라이더), 몽골어 등 미지원 언어 폴백 안내.
- 기록 보존/자동파기 정책 UI(`deskLog` 열람·만료삭제).
- 키오스크 보강(Wake Lock·권한 유지).
