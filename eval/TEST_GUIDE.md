# KAC 통역 정확성 테스트 가이드

실제 사람이 발화한 **7개국어 질문·답변**으로 STT(원문 인식)와 번역을 **분리 측정**하고,
오류를 유형별로 뽑아 개선 우선순위를 정하는 전체 절차.

- 대상 언어: 영어·중국어·일본어·스페인어·프랑스어·포르투갈어·아랍어
- 측정 축:
  - **질문(q)** — 외국인 발화 → 한국어 (외국어 STT + 외→한 번역)
  - **답변(a)** — 한국인 발화 → 외국어 (한국어 STT + 한→외 번역)

---

## ⭐ 표준 절차 — 안내데스크 모드 그대로 쓰기 (별도 도구 없음)

평소 운영과 동일하게 **안내데스크 모드로 응대를 진행**하고, 남은 응대 로그를 JSON 으로 내려받아 채점합니다.
시나리오 번호를 손으로 붙일 필요가 없습니다 — 채점기가 발화 내용으로 **자동 매칭**합니다.

1. **응대 진행** — 데스크 세션에서 손님 태블릿으로 언어 선택 → 워드파일(`wordfiles/KAC_평가_<언어>.docx`)의
   문장을 순서대로 낭독(질문=외국인·원어 굵은줄, 답변=안내원·한국어 굵은줄). 30개를 다 읽고 응대 종료.
   - 순서가 섞이거나 일부를 건너뛰어도 됩니다(자동 매칭이라 무관).
2. **평가 JSON 내려받기** — 관리자 → **로그** → 해당 응대 클릭(전문 열림) → **[평가 JSON 내려받기]**.
   `records-<세션>-<회차>.json` 이 저장됩니다(발화별 인식 원문·번역·방향·언어 포함).
3. **채점** —
   ```bash
   node eval/score.mjs records-xxx-0.json --auto                  # OpenAI 채점(과금)
   node eval/score.mjs records-xxx-0.json --auto --dry            # 무료(CER만)
   ```
   `--auto` 가 각 발화를 30개 시나리오 정답과 유사도로 대조해 자동 배정하고,
   대본에 없는 잡담·인사말은 "미매칭"으로 제외해 알려줍니다.
   여러 회차/언어를 모아서 채점하려면 records 파일들의 `records` 배열을 이어 붙이면 됩니다.

아래는 형식·지표·수동 절차의 상세 설명입니다.

## 0. 준비물

| 항목 | 파일/위치 | 비고 |
|---|---|---|
| 정답셋 | `eval/dataset.json` | 30개 시나리오(난이도 하·중·상 각 10) |
| 낭독 카드 | `eval/wordfiles/KAC_평가_<언어>.docx` | 언어별 7종, 인쇄해서 사용 |
| 채점기 | `eval/score.mjs` | Node 실행 |
| 정답셋 원본 | `eval/gen_dataset.py` | 문구 수정 시 여기서 → 재생성 |

> 정답셋·워드파일을 고칠 때는 `eval/gen_dataset.py` 를 수정하고 `python3 eval/gen_dataset.py` 재실행.
> (dataset.json 과 워드파일이 함께 갱신됨)

**검수 필수** — es/fr/pt/ar 번역과 파라타항공 표기는 초안입니다. 실측 전 원어민 검수로 확정하세요.
정답이 틀리면 측정 전체가 틀어집니다.

## 1. 화자·환경 세팅

1. **화자 섭외** — 질문은 외국인(가능하면 **비원어민 억양** 포함) 언어당 2~3명,
   답변은 한국인 안내원 2~3명. 한 명만 녹음하면 그 사람에게 과적합된 결과가 나옵니다.
2. **환경** — 조용한 방이 아니라 **실제 데스크 소음·마이크 거리**에서 진행. 현장 성능이 목표입니다.
3. **낭독 카드 배포** — 해당 언어의 `KAC_평가_<언어>.docx` 를 인쇄/화면에 띄움.
   - 질문 카드: **원어 → 영어 → 한국어** 순. 외국인은 **굵은 줄(원어)** 을 읽습니다.
   - 답변 카드: **한국어 → 영어 → 원어** 순. 안내원은 **굵은 줄(한국어)** 을 읽습니다.
   - 난이도 하 → 중 → 상 순서로 배열돼 있습니다.

---

## 2. 발화 수집 (데스크 모드)

각 시나리오를 순서대로 발화하고, 시스템이 인식한 **원문**과 **번역**을 기록합니다.

1. 관리자로 로그인 → **데스크 안내** 세션 생성(또는 기존 데스크 사용).
2. 손님(외국인) 태블릿에서 해당 **언어 선택** → 통역 시작.
3. **질문 낭독** — 외국인이 카드의 굵은 줄(원어)을 읽음 → 화면에 인식 원문·한국어 번역 표시.
4. **답변 낭독** — 안내원이 대응 답변 카드의 굵은 줄(한국어)을 읽음 → 외국어 번역 표시.
5. 시나리오 번호(`scenario_id`)를 메모하며 30개를 진행. (언어별 반복)

> 팁: 한 언어를 한 세션에서 끝내면 로그가 세션 단위로 깔끔히 모입니다.

---

## 3. 결과를 records.json 으로 정리

관리자 → **로그**에서 해당 데스크 응대의 항목(각 항목 `source`=인식 원문, `texts`=번역)을 꺼내
아래 형식으로 옮깁니다. `scenario_id`·`direction`·`lang` 을 손으로 붙입니다.

```json
{ "records": [
  { "scenario_id": "gate-find", "direction": "q", "lang": "en",
    "stt": "How do I get to gate twenty seven",   // 시스템이 인식한 원문
    "mt":  "27번 게이트에 어떻게 가나요?",           // 시스템 번역
    "latency_ms": 1200, "speaker": "spk1" },        // (선택)
  { "scenario_id": "gate-find", "direction": "a", "lang": "en",
    "stt": "27번 게이트는 이 복도를 따라가면 오른쪽 끝에 있습니다.",
    "mt":  "Gate 27 is at the far right end if you follow this hallway." }
] }
```

- `direction`: `q`(질문, 외국어→한) / `a`(답변, 한→외국어)
- **CER 정답은 발화된 형태로**: 화자가 "gate twenty-seven"이라 말하면 정답셋의 원문도 그렇게 둬야
  숫자 표기(27 vs twenty-seven) 차이가 인식 오류로 잘못 잡히지 않습니다. (필요 시 gen_dataset.py 조정)
- 파일 경로 예: `eval/records.json`

## 4. 채점 실행

```bash
# (a) 형식·CER 만 — OpenAI 호출 없음, 무료
node eval/score.mjs eval/records.json --dry

# (b) 전체 채점 — OpenAI 적합성 심사까지 (.env 의 OPENAI_API_KEY 사용, 과금)
node eval/score.mjs eval/records.json --out eval/result.json

# 심사 모델 지정(기본 gpt-5-mini)
OPENAI_JUDGE_MODEL=gpt-5 node eval/score.mjs eval/records.json --out eval/result.json
```

산출 리포트:
- **방향·언어별** CER(원문 인식)·적합성(번역)·정상률
- **오류 귀속** 분포 — `stt`(인식탓) / `mt`(번역탓) / `stt_recovered`(인식오류를 번역이 복원)
- **번역 오류 유형** 히스토그램 (고유명사·숫자·게이트·편명·시간 …)
- **함정유형별 취약도**, **최악 사례 Top 8**(원문·번역·심사 사유 포함)
- `--out` 지정 시 발화별 상세 결과 JSON 저장

## 5. 결과 해석 — STT vs 번역 분리

| 인식(CER) | 번역(적합성) | 귀속 | 개선 방향 |
|---|---|---|---|
| 정상 | 나쁨 | **번역탓(mt)** | 용어집·번역 설정(alt 포함)·프롬프트 |
| 나쁨 | 나쁨 | **인식탓(stt)** | language_hints·마이크/게인·endpoint·용어(terms) |
| 나쁨 | 정상 | 인식오류를 번역이 복원 | (관찰만) |
| 정상 | 정상 | 정상 | — |

기준값: CER ≤ 0.10 → 인식 정상, 적합성 ≥ 4/5 → 번역 정상. (score.mjs 상단에서 조정 가능)

**항공사 약어 확인 포인트** — 중국인이 `国航`, 일본인이 `全日空`/`JAL` 로 말했을 때
① 그대로 인식되는지(CER), ② 한국어로 `중국국제항공`/`전일본공수`/`일본항공` 으로 나오는지(적합성).
안 되면 관리자 → 용어 설정 → 번역 설정의 해당 행 **'약어·구어'** 에 표기를 추가.

---

## 6. 개선 루프

1. **파일럿** — 영↔한 + 1개 언어부터 30문항 채점.
2. 오류 귀속·유형 집계 → **가장 큰 덩어리부터** 개선(용어집 / 약어(alt) / language_hints / 프롬프트).
3. **같은 records 조건으로 재측정**(A/B) → 개선폭 확인. 핵심은 동일 데이터셋 반복 측정.
4. 안정되면 나머지 언어로 확장.
