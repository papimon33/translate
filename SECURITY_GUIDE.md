# 보안 설정 가이드 (운영자용)

이 문서는 KAC Translator 를 안전하게 운영하기 위한 3가지 보안 기능의 개념과 설정 절차를 설명합니다.
모두 **환경변수**(.env 파일 또는 Render 대시보드의 Environment)로 켭니다.

---

## 1. HTTPS 강제 (FORCE_HTTPS)

**왜 필요한가:** http 로 접속하면 로그인 비밀번호·대화 내용이 네트워크에서 그대로 노출될 수 있습니다.
https 는 이 통신을 암호화합니다. HSTS 헤더는 이미 서버에 내장되어 있어, https 로 한 번 접속한
브라우저는 이후 자동으로 https 만 사용합니다.

**설정 방법:**
- **Render 에 배포한 경우** — 아무것도 할 필요 없습니다. Render 가 모든 접속에 https 를 자동 적용합니다.
- **직접 서버를 운영하는 경우** — `.env` 에 아래 한 줄을 추가하면 http 접속이 https 로 자동 이동됩니다.
  ```
  FORCE_HTTPS=1
  ```
  단, 서버 앞에 https 를 처리하는 프록시(nginx, Caddy 등)나 인증서(Let's Encrypt 무료 발급)가 먼저 있어야 합니다.
  가장 간단한 방법은 [Caddy](https://caddyserver.com) 를 앞단에 두는 것입니다 — 도메인만 있으면 인증서 발급·갱신이 전부 자동입니다.
- **로컬 개발(localhost)** 은 예외 처리되어 있어 그대로 쓰면 됩니다.

---

## 2. 관리자 2단계 인증 (2FA)

**왜 필요한가:** 관리자 비밀번호가 유출되어도, 로그인하려면 관리자 휴대폰의 인증 앱에 표시되는
6자리 코드가 추가로 필요해집니다. 표준 TOTP 방식이라 Google Authenticator, Microsoft Authenticator,
1Password 등 아무 인증 앱이나 사용할 수 있습니다.

**설정 절차 (5분):**
1. 휴대폰에 **Google Authenticator** 앱을 설치합니다.
2. KAC Translator 에 관리자로 로그인 → **관리자 → 시스템·보안** 탭 → **2FA 설정 시작**.
3. 화면의 QR 코드를 인증 앱으로 스캔합니다. (스캔이 안 되면 아래 '수동 입력 키'를 앱에 직접 입력)
4. 앱에 표시된 6자리 코드를 입력하고 **활성화**를 누릅니다.
5. 이후 관리자 로그인 시: ID·비밀번호 입력 → 인증 코드 입력란이 추가로 나타납니다.

**주의사항:**
- 휴대폰을 분실하면 로그인할 수 없습니다. 서버에 접근 가능하다면 `data/security.json` 파일을 삭제하면 2FA 가 해제됩니다.
- 여러 서버 인스턴스를 운영하거나 파일이 유지되지 않는 환경(Render 무료 티어는 재배포 시 파일이 초기화됨)에서는
  설정 완료 후 표시된 '수동 입력 키'를 환경변수로 옮기는 것이 안전합니다:
  ```
  ADMIN_TOTP_SECRET=<수동 입력 키>
  ```
  환경변수가 설정되면 파일보다 우선하며, 관리자 화면에서의 변경은 잠깁니다.

## 3. 저장 데이터 암호화 (DATA_KEY)

**왜 필요한가:** 대화 기록·사용자 계정이 서버의 `data/*.json` 파일에 저장됩니다. 서버 디스크나 백업이
유출되면 내용이 그대로 읽힙니다. DATA_KEY 를 설정하면 이 파일들이 AES-256-GCM 으로 암호화되어,
키 없이는 읽을 수 없게 됩니다.

**설정 절차:**
1. 강한 무작위 키를 하나 생성합니다. 터미널에서:
   ```
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```
2. `.env`(또는 Render Environment)에 추가:
   ```
   DATA_KEY=<위에서 생성한 값>
   ```
3. 서버를 재시작합니다. 기존 평문 파일은 그대로 읽히고, **다음 저장부터 자동으로 암호문으로 바뀝니다**
   (별도 마이그레이션 불필요).

**반드시 지킬 것:**
- **키를 잃어버리면 데이터를 복구할 수 없습니다.** 키를 비밀번호 관리자(1Password 등)에 보관하세요.
- 키를 바꾸면 기존 암호문을 읽을 수 없습니다. 바꿔야 한다면: 키 제거 → 서버 기동(평문 저장으로 복귀) →
  새 키 설정 순서로 진행하세요.
- `.env` 파일은 git 에 올라가지 않지만(이미 .gitignore), 서버 접근 권한 관리에 주의하세요.

**MongoDB(Atlas) 를 쓰는 경우:** 파일 암호화는 파일 모드에만 적용됩니다. MongoDB Atlas 는
저장 데이터 암호화(Encryption at Rest)가 **기본으로 켜져 있어** 별도 설정이 필요 없습니다.
추가로 강화하려면 Atlas 콘솔에서 네트워크 접근(IP Allowlist)을 서버 IP 로 제한하세요.

## 현재 상태 확인

관리자 → **시스템·보안** 탭에서 세 항목(암호화·2FA·HTTPS 강제)의 켜짐/꺼짐 상태를 한눈에 볼 수 있습니다.

## 운영(production) 기동 게이트

`NODE_ENV=production` 에서는 다음이 미설정이면 **서버가 기동을 거부**합니다(보안 사고 예방):
- `AUTH_SECRET` 미설정 — 로그인 쿠키 서명 키가 비밀번호에서 파생되어 위조 가능해지므로 차단
- `ADMIN_PASSWORD` 미설정 또는 기본값(admin) — 관리자 탈취 직결이므로 차단

검증 환경에서 의도적으로 건너뛰려면 `ALLOW_INSECURE=1` 을 설정하세요(운영에서는 금지).
추가로 `PUBLIC_HOST=서비스도메인` 을 설정하면 FORCE_HTTPS 리다이렉트가 그 도메인으로 고정되어
호스트 헤더 스푸핑을 통한 오픈 리다이렉트가 차단됩니다.
로그인 쿠키는 발급 후 30일이 지나면 자동 만료됩니다(탈취 쿠키 영구 유효 방지).
